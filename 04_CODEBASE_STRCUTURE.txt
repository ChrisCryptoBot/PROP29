## Naming Conventions (ACTUAL IMPLEMENTATION - 100% ACCURATE)

### File and Directory Naming Patterns
```
Current Implementation Patterns:
- Python files: snake_case (auth_service.py, incident_service.py)
- TypeScript/React files: PascalCase for components (Dashboard.tsx, Layout.tsx)
- JavaScript files: camelCase for utilities (modalConfig.js)
- CSS files: kebab-case (ModulePanel.css, Dashboard.css)
- Configuration files: descriptive names (tailwind.config.js, tsconfig.json)
- Documentation files: descriptive names with extensions (.md, .txt)
- Service directories: snake_case (ai_ml_service/, security_service/)
- Component directories: PascalCase (Layout/, Admin/, UI/)
```

### Code Organization Patterns (ACTUAL IMPLEMENTATION - 100% ACCURATE)
```
Python Service Pattern:
service_name/
├── __init__.py                         # Package initialization
├── {service_name}.py                   # Main service file
└── subdirectories/                     # Service-specific subdirectories
    ├── __init__.py                     # Subdirectory initialization
    └── specialized_service.py          # Specialized functionality

TypeScript Component Pattern:
{ComponentName}/
├── {ComponentName}.tsx                 # Component implementation
├── {ComponentName}.test.tsx            # Component tests
├── {ComponentName}.css                 # Component styles (if separate)
└── index.ts                            # Component export (if needed)

Page Component Pattern:
pages/
├── {PageName}.tsx                      # Page component
├── {PageName}.css                      # Page styles (if separate)
└── modules/                            # Module-specific pages
    ├── {ModuleName}.tsx                # Module page
    └── {ModuleName}Auth.tsx            # Module authentication
```
