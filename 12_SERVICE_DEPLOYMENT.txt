Template 2: Technical Requirements – Specifies technical constraints such as performance, scalability, and security needs. This guides the AI in selecting appropriate technologies and designing within defined boundaries.  

Template 3: System Architecture – Provides a high-level blueprint of how the system’s components interact. This gives the AI a clear picture of the system’s structure, enabling it to align code generation with the intended design.

Role in AI Context: By starting with these templates, the AI gains a foundational understanding of the project’s objectives and technical framework, ensuring all subsequent outputs are purpose-driven and technically sound.
Phase 2: Templates 4-6 establish HOW to build
This phase focuses on the practical standards and configurations needed for implementation.  
Template 4: Codebase Structure – Defines the file and directory organization, ensuring the AI knows where to place generated code and how to maintain a scalable, logical structure.  

Template 5: Development Standards – Sets coding conventions, testing requirements, and quality benchmarks. This ensures the AI produces consistent, maintainable, and professional-grade code.  

Template 6: Environment Configuration – Details the setup for development, staging, and production environments, including database connections and secret management. This prepares the AI to generate code that integrates seamlessly across deployment stages.

Role in AI Context: With the foundation established, these templates provide the AI with clear guidelines on coding practices and environment setups, enabling it to produce functional and deployable code.

Role in AI Context: These templates equip the AI with tested patterns and examples, reducing errors and ensuring the generated code aligns with real-world requirements and best practices.
